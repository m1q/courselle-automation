## Project Overview

Courselle Automation is a Node.js project that converts HTML slides to PNG images using Playwright. It supports multiple content projects (sub-projects) each containing HTML slides and an export script. The primary use case is batch exporting slides for social media content.

Both contain the same information about setup and usage.

## Common Commands

### Root-level npm scripts (from package.json)
- `npm run export` вЂ“ Export a single slide using `src/export.js` (basic example)
- `npm run export-all` вЂ“ Run `scripts/export-all-projects.js` to export all subвЂ‘projects (note: script name mismatch вЂ“ `export-all` in package.json calls `scripts/export-all.js` which does not exist; actual script is `scripts/export-all-projects.js`)
- `npm run clean` вЂ“ Remove files from `output/` (not used by default)
- `npm run setup` вЂ“ Install dependencies

### Project control script (`scripts/projectctl.sh`)
The main orchestration script. Usage:
- `./scripts/projectctl.sh prepare <project-name>` вЂ“ Create a new project from HTML files in Windows Downloads folder
- `./scripts/projectctl.sh prepare <project-name> --single` вЂ“ Prepare a singleвЂ‘slide project
- `./scripts/projectctl.sh prepare <project-name> --empty` вЂ“ Create an empty project structure
- `./scripts/projectctl.sh export <project-name>` вЂ“ Export all slides of a specific project
- `./scripts/projectctl.sh export <project-name> --first` вЂ“ Export only the first slide
- `./scripts/projectctl.sh export-all` вЂ“ Export all existing projects sequentially
- `./scripts/projectctl.sh doctor` вЂ“ Check system and project health

### Individual project export
Each subвЂ‘project has its own `export-all.js`. To export a single project:
```bash
cd projects/<project-name>
npm install   # installs Playwright locally (optional)
node export-all.js
```

### Legacy scripts (superseded by projectctl.sh)
- `scripts/export_project.sh` вЂ“ Legacy export script for a single project
- `scripts/prepare_project.sh` вЂ“ Legacy preparation script
- `scripts/prepare_inorder_project.sh` вЂ“ Legacy ordered preparation script

### How Export Works
1. Playwright launches a headless Chromium browser.
2. Each HTML file is loaded via `file://` URL.
3. The script waits for fonts to be ready and adds a short delay (default 700вЂЇms).
4. It looks for an element with class `slide`; if found, screenshots that element; otherwise screenshots the whole page.
5. PNGs are saved to the projectвЂ™s `png/` folder with the same base name as the HTML file.

### Configuration
Environment variables (used by `projectctl.sh` and generated `export-all.js`):
- `VIEWPORT_WIDTH` вЂ“ Viewport width (default: 1080)
- `VIEWPORT_HEIGHT` вЂ“ Viewport height (default: 1350)
- `WAIT_MS` вЂ“ Milliseconds to wait before screenshot (default: 700)
- `EXPORT_FIRST_ONLY` вЂ“ Set to `1` to export only the first slide
- `PROJECTS_ROOT` вЂ“ Path to projects directory (default: `$REPO_ROOT/projects`)
- `WIN_DOWNLOADS` вЂ“ Path to Windows Downloads folder in WSL (default: `/mnt/c/Users/user/Downloads`)

These can be overridden by setting the variables before running the script or editing the generated `export-all.js`.

## Notes for Development

- **HTML slide structure**: Each slide should be contained in an element with class `slide` for optimal screenshot cropping. If no `.slide` element exists, the entire page is captured.
- **File naming**: Slides must be named `slide1.html`, `slide2.html`, вЂ¦ (numbers can be any order; they are sorted numerically). The script also supports `preview.html` and `preview (N).html` formats (used by Canva exports).
- **Playwright installation**: The root `package.json` includes Playwright as a dev dependency. Each subвЂ‘project can also have its own `package.json` (created by `projectctl.sh`). The `projectctl.sh` script ensures Playwright is installed in the root and browsers are available.
- **Windows integration**: The preparation scripts expect HTML files in `/mnt/c/Users/user/Downloads` (Windows downloads folder mounted in WSL). Adjust `WIN_DOWNLOADS` in `projectctl.sh` if needed.
- **Project name sanitization**: `projectctl.sh` sanitizes project names by replacing characters `< > : " / \\ | ? *` with hyphens and trimming spaces.
- **Git ignored**: `node_modules/`, `output/`, `projects/*/png/`, `projects/*/node_modules/`, and `.playwright/` are ignored. PNG files inside `projects/*/html/` are kept (exception).

## Troubleshooting

- **Playwright browser not installed**: Run `npx playwright install chromium` in the repo root.
- **No HTML files found in Downloads**: Ensure `WIN_DOWNLOADS` points to the correct Windows Downloads folder mounted in WSL.
- **Export fails with font loading errors**: Increase `WAIT_MS` environment variable to allow more time for fonts to load.
- **Package.json script mismatch**: The `npm run export-all` script points to `scripts/export-all.js` which does not exist. Use `node scripts/export-all-projects.js` directly or update package.json.
- **Project with special characters**: `projectctl.sh` will sanitize names, but avoid using problematic characters in project names.
