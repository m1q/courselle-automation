## Project Overview

Courselle Automation is a Node.js project that converts HTML slides to PNG images using Playwright. It supports multiple content projects (sub-projects) each containing HTML slides and an export script. The primary use case is batch exporting slides for social media content.

### Documentation
- `README.md` – Arabic documentation
- `README_EN.md` – English documentation
Both contain the same information about setup and usage.

## Common Commands

### Root-level npm scripts (from package.json)
- `npm run export` – Export a single slide using `src/export.js` (basic example)
- `npm run export-all` – Run `scripts/export-all-projects.js` to export all sub‑projects
- `npm run clean` – Remove files from `output/` (not used by default)
- `npm run setup` – Install dependencies

### Project control script (`scripts/projectctl.sh`)
The main orchestration script. Usage:
- `./scripts/projectctl.sh prepare <project-name>` – Create a new project from HTML files in Windows Downloads folder
- `./scripts/projectctl.sh prepare <project-name> --single` – Prepare a single‑slide project
- `./scripts/projectctl.sh prepare <project-name> --empty` – Create an empty project structure
- `./scripts/projectctl.sh export <project-name>` – Export all slides of a specific project
- `./scripts/projectctl.sh export <project-name> --first` – Export only the first slide
- `./scripts/projectctl.sh export-all` – Export all existing projects sequentially
- `./scripts/projectctl.sh doctor` – Check system and project health

### Individual project export
Each sub‑project has its own `export-all.js`. To export a single project:
```bash
cd projects/<project-name>
npm install   # installs Playwright locally (optional)
node export-all.js
```

## Architecture

### Directory Structure
```
courselle-automation/
├── src/                    # Core source (example export script and HTML template)
├── projects/              # Content sub‑projects (7-hapit, Framex, ads, anythings, etc.)
│   ├── <project-name>/
│   │   ├── html/          # HTML slide files (slide1.html, slide2.html, …)
│   │   ├── png/           # Output PNG images (generated)
│   │   ├── export-all.js  # Project‑specific export script (generated by projectctl)
│   │   └── package.json   # Optional local dependencies
├── scripts/               # Automation scripts
│   ├── projectctl.sh      # Main orchestration script
│   ├── export-all-projects.js # Export all projects from root
│   ├── export_project.sh      # Legacy export script
│   ├── prepare_project.sh     # Legacy preparation script
│   └── prepare_inorder_project.sh
├── output/                # Default output folder (not used by sub‑projects)
└── package.json          # Root dependencies (Playwright, Prettier)
```

### How Export Works
1. Playwright launches a headless Chromium browser.
2. Each HTML file is loaded via `file://` URL.
3. The script waits for fonts to be ready and adds a short delay (default 700 ms).
4. It looks for an element with class `slide`; if found, screenshots that element; otherwise screenshots the whole page.
5. PNGs are saved to the project’s `png/` folder with the same base name as the HTML file.

### Configuration
Environment variables (used by `projectctl.sh` and generated `export-all.js`):
- `VIEWPORT_WIDTH` – Viewport width (default: 1080)
- `VIEWPORT_HEIGHT` – Viewport height (default: 1350)
- `WAIT_MS` – Milliseconds to wait before screenshot (default: 700)
- `EXPORT_FIRST_ONLY` – Set to `1` to export only the first slide
- `PROJECTS_ROOT` – Path to projects directory (default: `$REPO_ROOT/projects`)
- `WIN_DOWNLOADS` – Path to Windows Downloads folder in WSL (default: `/mnt/c/Users/user/Downloads`)

These can be overridden by setting the variables before running the script or editing the generated `export-all.js`.

## Notes for Development

- **HTML slide structure**: Each slide should be contained in an element with class `slide` for optimal screenshot cropping. If no `.slide` element exists, the entire page is captured.
- **File naming**: Slides must be named `slide1.html`, `slide2.html`, … (numbers can be any order; they are sorted numerically).
- **Playwright installation**: The root `package.json` includes Playwright as a dev dependency. Each sub‑project can also have its own `package.json` (created by `projectctl.sh`). The `projectctl.sh` script ensures Playwright is installed in the root and browsers are available.
- **Windows integration**: The preparation scripts expect HTML files in `/mnt/c/Users/user/Downloads` (Windows downloads folder mounted in WSL). Adjust `WIN_DOWNLOADS` in `projectctl.sh` if needed.
- **Git ignored**: `node_modules/`, `output/`, `projects/*/png/`, `projects/*/node_modules/`, and `.playwright/` are ignored.
